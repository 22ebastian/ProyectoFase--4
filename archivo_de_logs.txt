        # Garantiza que el sistema no falle catastróficamente en el último segundo.
        Logger.registrar_evento("ERROR", f"Fallo crítico final: {str(e)}")
        print(f"Error imprevisto en Simulación 10: {e}")
        
    else:
        # Confirmación de que el sistema alcanzó el final del ciclo de vida de forma estable.
        # Este es el indicador clave para el 'Nivel Alto' de la rúbrica [5].
        mensaje_final = "Simulación 10 exitosa: El sistema ha demostrado estabilidad total tras 10 pruebas."
        Logger.registrar_evento("INFO", mensaje_final)
        print(mensaje_final)
        
    finally:
        # Bloque de clausura que confirma al usuario que los datos están seguros en el log [1].
        print("--- Proceso de Simulaciones de Software FJ Finalizado ---")
        print("Verifique el archivo 'archivo_de_logs.txt' para ver el historial completo.")

Datos inválidos: Situaciones donde se ingresa información que no cumple con el formato o tipo esperado, como registros de clientes con datos erróneos

Parámetros faltantes: Intentos de crear objetos o instanciar servicios sin suministrar toda la información obligatoria requerida por el diseño de las clases

Servicios no disponibles: Peticiones de servicios que el sistema no puede procesar en el momento, garantizando que el programa no falle al encontrarse con un servicio inactivo o inexistente

Cálculos inconsistentes: Errores detectados durante los procesos de cálculo de costos, especialmente cuando se aplican métodos sobrecargados con impuestos, descuentos o parámetros opcionales

Operaciones no permitidas: Acciones que el sistema debe bloquear por seguridad o lógica de negocio, evitando que se comprometa la integridad de la aplicación

Intentos de reserva incorrectos: Fallos durante el proceso de gestión de reservas, incluyendo reservas fallidas que deben ser capturadas de manera controlada
